# AGENTS Instructions

## Commit Message Rules

- Commit messages must be written in English.
- Use a type prefix at the start of every commit message.
- Recommended format:

```text
<type>: <short summary>
```

- Keep the summary concise and action-oriented.
- Use lowercase for the type prefix.
- Do not use vague messages such as `update`, `fix stuff`, or `misc changes`.

### Allowed Commit Types

- `feat`: a new feature
- `fix`: a bug fix
- `refactor`: code restructuring without changing intended behavior
- `docs`: documentation changes
- `test`: adding or updating tests
- `chore`: maintenance or tooling changes
- `perf`: performance improvements
- `build`: build system or dependency packaging changes
- `ci`: CI/CD configuration changes
- `style`: formatting or non-functional code style changes
